## 硬件数据单一事实源（2026-05-12 11:45）

### 根因
- 旧：`admin.html` 保存时写入 `localStorage`（`pcbuilder_hardware_override`），而线上装机走 `/api/hardware/:category` 读 `data/hardware-v2.json`，两条路完全不通
- 后台改了 → 前台用户看不到

### 修复
- **新增 API**：
  - `GET /api/admin/hardware/all` → 读取 `hardware-v2.json` 全量数据（供 admin 加载）
  - `POST /api/admin/hardware/save` → 全量写入 `hardware-v2.json`（原子替换 items 数组）
  - `DELETE /api/admin/hardware/:id` → 从 `hardware-v2.json` 删除单条
- **admin.html**：`loadData()` 优先从 `/api/admin/hardware/all` 加载；`保存修改` 改为 `POST /api/admin/hardware/save`
- **前台**：`/api/hardware/:category` 读同一 `hardware-v2.json`，不变

### 硬件真相源
- **路径**：`/tmp/pc-builder/data/hardware-v2.json`
- **结构**：`{ meta, items: [{ id, name, specs, price, brand, category, ... }] }`
- **12 品类 key**：cpu, gpu, motherboard, ram, storage, cooling, psu, case, monitor, keyboard, mouse, headset
